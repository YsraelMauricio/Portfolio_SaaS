# DATA_MODEL.md — Data Model

**Complements `PROJECT_SPEC.md` v3.1.** Engine: PostgreSQL. This document is the source of truth for table and field names — `API_SPEC.md` and `ARCHITECTURE.md` must use these exact names, never varying them.

> **Naming convention:** all tables — including business-domain ones — use **English**, consistent across the whole codebase and with the Laravel/Next.js ecosystem conventions (Spatie packages, Sanctum, standard tutorials/docs). No Spanish table or column names anywhere. User-facing content (blog posts, portfolio descriptions) is translated via dedicated `*_translations` tables — that is the only place language varies, and it varies by *data*, not by *schema*.

### `contracts`
| Field | Type | Notes |
|---|---|---|
| id | bigint PK | |
| project_id | FK → projects.id | |
| quote_snapshot | jsonb | **Frozen copy** at generation time, always this exact shape regardless of source: `{ "product_type_name": string, "price_usd": decimal, "estimated_days_min": int, "estimated_days_max": int, "modifiers": [string], "technologies": [string] }`. Never recalculated or live-referenced against `quotes` after the fact. When `project.quote_id` is present, these six fields are copied automatically from that quote. When it's null (the custom-project path), there's no `quotes` row to copy from — the admin supplies this **same shape** by hand via `quote_snapshot_override` in the `POST /admin/contracts` body (`API_SPEC.md` §3). | generated_at, approved_by_admin_at, sent_at, signed_at, cancelled_at | timestamp, nullable | |
| pdf_media_id | FK nullable → media.id | Stored via Spatie Media Library (`contract_pdf` collection) — same mechanism as every other file in the system, rather than a bare URL string |
| is_test | boolean, default false | |

### `payments`

| id | bigint PK | |

## 9. Critical indexes and constraints (non-negotiable)

- `users.email` — unique.
- `payments.provider_transaction_id` — unique when not null (basis of idempotency).
- `blog_post_translations` (`blog_post_id`, `locale`) — composite unique.
- `portfolio_project_translations` (`portfolio_project_id`, `locale`) — composite unique.
- Index on `quotes.is_test`, `projects.is_test`, `payments.is_test`, `contracts.is_test` — the BI dashboard filters on this in every query.
- Index on `projects.status` — queried constantly for the pipeline and the "next available start date" calculation.
